Overview

This is an interactive showcase application that demonstrates Design System components through a clicker counter app. The app features input controls in the sidebar and visual components in the main content area that update in real-time based on your interactions.

This showcase demonstrates:

  • How to use Design System components in an interactive application
  • Component integration and state management
  • Real-time updates between input controls and visual display
  • Component variants: buttons (primary, secondary, danger), boxes/cards, tags (success, error), dropdowns, and inputs

Sidebar Controls

The sidebar contains input controls that affect the counter:

Counter Label

Use the text input field to change the label displayed above the counter value. Type any text you want, and it will update immediately in the main display area.

Increment Amount

Select the increment amount from the dropdown menu. You can choose:

  • 1 - Increment/decrement by 1
  • 5 - Increment/decrement by 5
  • 10 - Increment/decrement by 10
  • 25 - Increment/decrement by 25

The selected increment amount is displayed in the settings panel below the counter.

Action Buttons

  • Increment (Primary button) - Increases the counter by the selected increment amount
  • Decrement (Secondary button) - Decreases the counter by the selected increment amount
  • Reset (Danger button) - Resets the counter to 0

Main Display Area

The main content area shows visual components that update based on your interactions:

Counter Display

The counter value is displayed prominently in a card box component. The label above the counter updates when you change it in the sidebar.

Status Tags

Status tags automatically update based on the counter value:

  • Active (Primary tag) - Shown when counter is at 0
  • Positive (Success tag) - Shown when counter is greater than 0
  • Negative (Error tag) - Shown when counter is less than 0

Action Buttons

Action buttons in the display area mirror the sidebar controls. You can use either set of buttons to control the counter.

Settings Display

The settings panel shows the current label and increment amount in a read-only card component, providing a clear view of the current configuration.

Design System Components Used

This app demonstrates the following Design System components:

Buttons

  • Primary Button - Used for the main increment action
  • Secondary Button - Used for the decrement action
  • Danger Button - Used for the reset action

Boxes/Cards

  • Card Box - Used for the counter display and settings panel

Tags

  • Primary Tag - Shows "Active" status
  • Success Tag - Shows "Positive" status
  • Error Tag - Shows "Negative" status

Dropdown

  • Dropdown Component - Used to select the increment amount

Input

  • Text Input - Used for the counter label input
